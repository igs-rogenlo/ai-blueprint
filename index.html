理解 AI Agent 的三個核心概念
A

Agent（代理人）

你的專屬 AI 助理

你給目標，它自己拆解步驟、選擇工具、完成任務。不只是聊天，而是主動執行。

M

MCP（萬能插頭）

Model Context Protocol

讓 Agent 可以安全連接各種資料庫和工具。像 USB-C 一樣，一個插頭搞定所有裝置。

S

Skill（技能）

Agent 學會的招式

每個 Skill 解決一個特定問題。像手機 App，查營收是一個 Skill，生成圖表是另一個。

傳統工具 vs AI Agent

從被動回答到主動執行

傳統 AI 工具

你問一個問題
→ AI 回答一段文字
→ 結束

像 Google 搜尋：
你查、你讀、你整理

AI Agent

你說出目標
→ Agent 自己查資料
→ 連資料庫
→ 產圖表
→ 寫報告

像真人助理：交代任務，全部搞定

OpenClaw 框架

我們用來打造和管理所有 Agent 的基礎架構

什麼是 OpenClaw？

OpenClaw 是我們自建的 Agent 管理框架，提供統一的方式來：

🤖 定義 Agent — 透過 AGENTS.md 設定每個 Agent 的任務和工具
🔌 串接 MCP — 統一管理資料庫和工具連接
⏰ Cron 排程 — 設定 Agent 定時自動執行任務
🧠 記憶系統 — LanceDB 語義記憶，Agent 能記住上下文
🌐 Browser 自動化 — ocb 指令操控瀏覽器
📨 Telegram 整合 — 透過 Cloudflare Workers Proxy 推送訊息

⏰ Cron 排程功能

Agent 不只能對話互動，還能定時自動執行：

📊 數據 Agent — 每天早上自動產出數據日報
📋 pm-tracker — 每天自動掃描專案狀態產出健康報表
📨 自動推送到 Telegram，不需人工觸發
