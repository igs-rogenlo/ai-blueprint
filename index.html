A

Agent（代理人）

你的專屬 AI 助理。

你只要說「幫我查上週營收做成圖表」，Agent 就會自己去查資料庫、做好圖表、傳到你的 Telegram。

跟 ChatGPT 的差別：ChatGPT 只能聊天，Agent 能實際去做事

M

MCP / API（連接管道）

Agent 存取公司系統的安全通道。

Agent 要查數據就連到資料庫、要看專案進度就連到 Redmine。MCP 和 API 就是這些連接的管道。

隱私保障：模型訓練已關閉，公司資料不會被用於 AI 訓練，資料不出公司

S

Skill（技能）

教會 Agent 的固定工作流程。

就像教新人「每天早上要做哪些事」，教一次之後，Agent 每次都會照著這個流程做。

例：「查營收做圖表」是一個 Skill，「掃描專案產報表」是另一個

三者的關係：Skill 定義 Agent「會做什麼」，MCP/API 決定 Agent「能連到哪裡」。你下指令 → Agent 選擇 Skill → 透過 MCP/API 去系統拿資料 → 把結果交給你。

給技術人員參考

MCP vs API 的差異 & 各 Agent 的連接細節

MCP vs API

MCP（常駐連線）

工具以 MCP Server 形式常駐運行，Agent 隨時呼叫，即時雙向通道

API（請求式呼叫）

傳統 REST API，發送請求等待回應，不需常駐 Server

Skill 與工作流

在 AGENTS.md 定義的固定 SOP，Agent 透過 LanceDB 語義記憶儲存後可重複執行。概念上等同於 Claude Code 的 Skill（.claude/skills/*.md），差異在於 Skill 是獨立模組可跨 Agent，工作流綁定在特定 Agent。

各 Agent 連接明細

bq-research

MCP: MSSQL MCP: LanceDB API: BigQuery API: QuickChart API: Telegram

pm-tracker

API: PM Tracker API: Redmine API: Mantis Browser: SharePoint API: Telegram
